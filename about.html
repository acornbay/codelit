CodeHilite is a free, browser-based tool that turns your code into gorgeous, shareable images — in seconds, with no account required, and nothing to install.

What is CodeHilite?

CodeHilite is a syntax-highlighted code snippet image generator built for developers who want to share their work without it looking like a raw terminal dump. Paste your code, pick a theme, adjust the layout, and export a crisp image ready for Twitter, LinkedIn, GitHub READMEs, blog posts, presentations, or anywhere else you share code.

Unlike screenshots of your IDE, CodeHilite gives you complete control over every visual detail — theme, font, font size, padding, corner radius, background colour or gradient, line numbers, and the macOS-style titlebar. The result is clean, intentional, and consistent regardless of which operating system, IDE, or configuration you use day-to-day.

Everything runs entirely in your browser. Your code is never transmitted to any server — ours or anyone else's. There's no database, no backend, and no account to create. Open the app, create your image, close the tab. That's the entire workflow.

Features

Syntax highlighting for 30+ languages

CodeHilite uses highlight.js — one of the most widely used syntax highlighting libraries in the world — to automatically detect and highlight your code's language. Auto-detection works by analysing the structure of your code and matching it against language patterns. For most languages, you can paste code and get accurate highlighting immediately. For ambiguous snippets (e.g. short JSON fragments that could be several things), you can override the auto-detection with the manual language selector.

Full layout control

Every visual parameter is adjustable. Font family (JetBrains Mono, Fira Code, Source Code Pro, and others), font size (8px to 24px), line height, padding (the space around the code inside the card), border radius (from sharp corners to fully rounded), drop shadow intensity, background colour, and background gradient. The live preview updates in real time as you adjust settings, so you see exactly what your export will look like before you click export.

Line highlighting

Draw attention to specific lines with a subtle background highlight. Click any line number in the preview to toggle its highlight state. Highlighted lines get a distinct background tint that draws the eye without obscuring the code. This feature is particularly valuable for tutorial creators who want to visually indicate "this is the part we're talking about now" without adding external annotation arrows in an image editor.

Multiple export formats

Most code screenshot tools only export PNG. CodeHilite exports in five formats:

  • PNG — lossless, transparent-background-capable. Best for social media, web embeds, presentations, and GitHub READMEs.
  • JPEG — smaller file sizes at the cost of some sharpness. Rarely the best choice for code, but useful in specific size-constrained contexts.
  • PDF — vector-backed, prints crisply at any size. Best for documents, academic papers, client reports, and LinkedIn carousel posts.
  • HTML — a self-contained HTML file with selectable, copyable code and inline syntax highlighting. Best for web embedding and accessibility.
  • Plain text — raw code content, no formatting. Best for pasting into forums, issue trackers, or Slack.

File drag and drop

Drag any source file from your file system directly onto the CodeHilite editor. The tool reads the file, detects the language from the file extension, and populates the editor automatically. This is faster than opening your IDE, copying code, switching to the browser, and pasting — especially useful when you're creating a screenshot from an existing file rather than freshly written code.

Session snippet saving

Save named snippets within your browser session and switch between them using the snippet panel. This lets you set up a series of screenshots for a tutorial or blog post, then export them one by one with consistent settings — without needing to paste, clear, and paste again for each one.

Window titlebar

Toggle a macOS-style titlebar (with the red, yellow, and green traffic light buttons) at the top of the code card. Set a custom filename that appears in the titlebar — e.g. auth-middleware.ts or useAnalytics.tsx. This small detail adds significant context to code screenshots and gives them a polished, professional feel that plain code blocks lack.

How it's built

CodeHilite is a single, self-contained HTML file. There's no build pipeline, no bundler, no framework, and no server-side component whatsoever. The entire application ships in one file and runs in your browser using vanilla JavaScript and a handful of well-established open-source libraries loaded from a CDN.

This architecture was a deliberate choice. Single-file tools are easy to audit, easy to self-host, and work completely offline once the page has loaded (except for the CDN-loaded libraries). If you're working on a proprietary codebase and want zero risk of code leaving your machine, you can download the HTML file and open it locally — everything works identically.

highlight.js

Handles syntax analysis and token-level colour application for all 30+ supported languages. highlight.js is a mature, well-maintained library used in millions of websites worldwide. Its auto-detection algorithm analyses code structure to identify the language without requiring any configuration.

html2canvas

Renders the live preview DOM element to a pixel-accurate canvas, which is then encoded as PNG or JPEG for download. html2canvas works by "painting" HTML and CSS to a canvas element in the browser, producing a bitmap that matches the visual output exactly.

jsPDF

Handles PDF generation. When you export as PDF, jsPDF creates a PDF document and embeds the canvas render as an image. The result is a single-page PDF that can be printed at high resolution or embedded in other documents.

CSS Variables for theming

All colour values in CodeHilite are defined as CSS custom properties (variables). Switching themes is a matter of updating a handful of variable values — the rest of the layout and component styling adapts automatically. This makes adding new themes straightforward and ensures visual consistency across the entire application.

Privacy by architecture: Because CodeHilite has no backend, there is genuinely no mechanism for your code to reach any server. It's not a policy promise that could be violated — it's a technical fact. The application has no API endpoints, no database connections, and no telemetry calls. Your code is processed by your own browser's JavaScript engine and stays there.

Privacy and data handling

CodeHilite collects no personal data. There are no cookies, no analytics scripts, no user tracking, and no accounts. The only external network requests the application makes are to load fonts from Google Fonts and libraries from Cloudflare's CDN — standard requests that any website using these services makes, and which expose only your IP address to the CDN provider as part of normal HTTP operation.

Your code never leaves your browser. It is not sent to CodeHilite's servers (there are no CodeHilite servers to send it to). It is not logged. It is not cached. When you close or refresh the page, your session data — including any saved snippets — is gone.

Frequently asked questions

Is CodeHilite really free?
Yes, completely. There are no paid plans, no premium themes, no watermarks, and no export limits. Everything in the app is available to everyone, with no account required.
Does my code get sent to a server?
No. CodeHilite has no backend server. Your code is processed entirely by your own browser's JavaScript engine. The only outbound network requests are to Google Fonts and Cloudflare CDN to load fonts and libraries — your code is never part of any network request.
What's the maximum snippet length?
There is no enforced maximum. Practically, snippets above 150–200 lines produce very large images and can slow down the html2canvas render. For long code files, it's usually better to crop to the most important section rather than showing everything.
Can I use CodeHilite offline?
Partially. The CodeHilite page can load and run offline after the first visit (browsers cache it), but the CDN-loaded libraries (highlight.js, html2canvas, jsPDF) and Google Fonts require an internet connection on first load. If you need fully offline use, download the HTML file and replace the CDN links with locally hosted copies of the libraries.
Can I add CodeHilite to my own website?
CodeHilite is a single HTML file — you can host it on your own web server. The source is available and you're welcome to use it as a starting point. Attribution is appreciated but not required.
Why does the export look slightly different from the preview?
The preview renders using your browser's native HTML/CSS rendering engine. The export uses html2canvas, which reimplements CSS rendering in a canvas context. Minor differences (particularly in font rendering and complex CSS gradients) can occur. If you notice a significant difference, try adjusting the export scale setting or using a simpler background.
My language isn't in the list — can it be added?
CodeHilite uses highlight.js for language support. If your language is supported by highlight.js but not listed in the CodeHilite dropdown, let us know and we'll add it. If your language isn't supported by highlight.js, the code will still display but without syntax colouring.
How do I report a bug or request a feature?
Use the Contact page. All feedback is read personally and bug reports are prioritised. Feature requests are tracked and the most frequently requested ones make it into the app.

CodeHilite is built and maintained as a solo project, which means every feature request gets a personal read and every bug report gets taken seriously.
