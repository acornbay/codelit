How it works

Code to image in four steps

Export formats

The right format for every use case

Most code screenshot tools export PNG and stop there. CodeHilite gives you five formats so you always have exactly what you need.

.png
PNG — recommended
Lossless image with transparent background support. Perfect for Twitter, Instagram, LinkedIn, GitHub READMEs, Notion, and slide decks. Looks sharp at any display resolution.
.pdf
PDF — for documents
Crisp at any print size. Use for technical reports, academic papers, client proposals, or LinkedIn carousel posts (upload as a native PDF document).
.html
HTML — for web
Self-contained HTML file with inline syntax highlighting. Code is selectable and copyable. Best for embedding on websites where accessibility and copy-paste matter.
.jpg
JPEG — compressed
Smaller file sizes than PNG at the cost of some sharpness. Use only when file size is a hard constraint — for example, embedding in a large multi-image PDF document.
.txt
Plain text — raw code
Strips all formatting and returns just the code content. Use for pasting into Slack, GitHub issues, forum posts, or any context that applies its own syntax highlighting.

FAQ

Frequently asked questions

Is CodeHilite really completely free? ▼
Yes. There are no paid plans, no premium themes, no export limits, and no watermarks. Everything in the app is available to everyone without an account.
Does my code get sent to a server? ▼
No. CodeHilite has no backend server at all. Your code is processed by your own browser's JavaScript engine. The only outbound network requests are to Google Fonts and Cloudflare CDN for fonts and libraries — your code is never part of any network request.
What's the maximum snippet length? ▼
There's no enforced limit. Snippets over 150–200 lines produce very large images and can slow the render. For long files, cropping to the most interesting section almost always produces better results anyway.
Which export format should I use? ▼
PNG for almost everything — social media, web, slides, GitHub READMEs. PDF for documents and print. HTML for websites where selectable code matters. JPEG only if file size is a hard constraint. Plain text for copy-pasting into forums or Slack.
Can I use CodeHilite offline? ▼
Partially. After the first load, the page is cached by your browser. However, the CDN-loaded libraries and Google Fonts need an internet connection on first load. For fully offline use, download the HTML file and self-host the libraries.
My language isn't auto-detected correctly — what do I do? ▼
Use the Language dropdown in the sidebar to select your language manually. Auto-detection works by analysing code structure and can misidentify short snippets or less common languages. Manual selection always takes priority.
How do I highlight specific lines? ▼
Click any line number in the preview pane to toggle its highlight state. Highlighted lines get a subtle background tint that draws the eye without obscuring the code. Click again to remove the highlight.
Can I report a bug or request a feature? ▼
Yes — use the Contact page. All feedback is read personally. Bug reports are prioritised and feature requests are tracked.

Your code never leaves your browser

No cookies

All syntax highlighting, rendering, and export processing happens entirely inside your browser. Your code never travels over any network.
