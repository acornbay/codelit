How it works

Code to image in four steps

No account, no install, no watermark. Paste your code, customise the look, and export — in under 30 seconds.

01
📋
Paste your code
Type or paste into the editor, or drag and drop a source file. Language is detected automatically from code structure or file extension.
02
🎨
Choose your style
Pick from 16+ syntax themes, adjust font, size, padding, corner radius, shadow, and background colour or gradient.

Export formats

The right format for every use case

Most code screenshot tools stop at PNG. CodeHilite gives you five formats so you always have exactly what you need.

.png
PNG — recommended
Lossless image with transparent background support. Best for Twitter, LinkedIn, Instagram, GitHub READMEs, Notion, and slide decks. Sharp at any display resolution.
.pdf
PDF — for documents
Prints crisply at any size. Use for technical reports, academic papers, client proposals, or LinkedIn carousel posts uploaded as a native PDF document.
.html
HTML — for web
Self-contained HTML with inline syntax highlighting. Code is selectable and copyable — ideal for embedding on websites where accessibility matters.
.jpg
JPEG — compressed
Smaller file sizes than PNG at the cost of some sharpness. Use only when file size is a hard constraint, such as embedding in a large multi-image PDF.
.txt
Plain text — raw code
Strips all formatting and returns just the code. Use for pasting into Slack, GitHub issues, forums, or anywhere that applies its own syntax highlighting.

FAQ

Frequently asked questions

Quick answers to the questions we hear most often.

Is CodeHilite really completely free? ▼
Yes. No paid plans, no premium themes, no export limits, no watermarks. Everything works for everyone without an account.
Does my code get sent to a server? ▼
No. CodeHilite has no backend server. Your code is processed by your own browser's JavaScript engine. The only external requests are to Google Fonts and Cloudflare CDN for fonts and libraries — your code is never part of any network request.
Which export format should I use? ▼
PNG for almost everything — social media, web, GitHub READMEs, slides. PDF for documents and print. HTML for websites where selectable text matters. JPEG only if file size is a hard constraint. Plain text for pasting into Slack, forums, or issue trackers.
How do I highlight specific lines? ▼
Use the "Highlight Lines" field in the sidebar. Enter comma-separated line numbers or ranges — for example 1,3,5-8 highlights lines 1, 3, and 5 through 8.
What is "Copy Formatted"? ▼
Copy Formatted copies your syntax-highlighted code as rich HTML. When you paste it into Microsoft Word, Google Docs, or Notion, the colours and monospace formatting are preserved — unlike a plain text paste.
Can I use CodeHilite offline? ▼
After the first page load, the app is cached by your browser and works offline. The CDN-loaded libraries and Google Fonts need a connection on first load. For fully offline use, self-host the HTML file with locally served copies of the libraries.
My language isn't auto-detected — what do I do? ▼
Use the language dropdown in the editor panel to select your language manually. Auto-detection can misidentify short snippets. The "Detect" button re-runs auto-detection on demand. Manual selection always takes priority.
How do I report a bug or request a feature? ▼
Use the Contact page. All feedback is read personally. Bugs are prioritised and feature requests are tracked — the most frequently asked ones make it into the app.
